# 中医国学 · 经典问答 — UI 设计系统文档 v5.0

> 本文档记录项目前端界面的设计规范、组件结构、视觉风格与交互逻辑，作为后续开发的一致参考依据。
> 最后更新：2026-07-03

---

## 一、设计哲学

- **温润如玉** — 整体调性如古书卷般温厚，不刺眼、不浮夸
- **呼吸感** — 每个区块之间的间距留给用户阅读的喘息空间
- **纸墨质感** — 米色底、深棕字，模拟宣纸与墨色的温润质感
- **克制优雅** — 不滥用动画，只在关键节点（消息出现、打字过程）使用细腻动效

## 三、组件规范

### 3.1 顶栏（Header）
- 渐变色背景（`#5a2d1a` → `#8B4513`）
- 白色文字，居中
- `position: sticky; top: 0` 固定在顶部
- 两层文字：标题 + 副标题
- `z-index: 20`

### 3.2 对话容器（Conversation View）
- 弹性布局，纵向滚动
- `scroll-behavior: smooth` 平滑滚动
- 自定义细滚动条（`4px` 宽，主题色滑块）
- 子元素间距 `clamp(16px, 2.5vw, 24px)`

### 3.3 用户消息气泡
- 右对齐（`align-items: flex-end`）
- 背景色 `--acc`（棕色），白色文字
- 圆角：`18px 18px 4px 18px`（右下角小折角）
- 最大宽度 `min(85%, 400px)`
- 使用 `--fs` 字体

### 3.4 AI 回复卡片
- 左对齐，全宽
- 白色卡片背景（`--card`），圆角 `20px`，柔和阴影
- 内边距 `clamp(18px, 3vw, 28px)`
- 内容从上到下：思考小窗 → 结构化回答 → 参考文献

### 3.5 思考过程小窗（Thinking Window）
- 米色背景（`#fcf9f4`），主题色边框
- 可折叠：点击标题栏展开/收起
- 默认**展开**状态
- 标题栏左侧🧠图标 + "思考过程"文字 + 右侧▼箭头
- 内容区使用楷体（`--ff-think`），`white-space: pre-wrap`
- 展开动画：`max-height` 过渡 + `cubic-bezier` 缓动

### 3.6 结构化内容块

#### 3.6.1 节标题（Section Header）
- Emoji 图标 + 粗体标题 + 下边框分割线
- 5 种预设颜色分类：
  - 💡 一句话精要 → 橙红（`.se`）
  - 📖 经典溯源 → 棕色（`.sc`）
  - 🧠 深度阐释 → 蓝色（`.sd`）
  - 🌿 生活践行 → 绿色（`.sl2`）
  - 📚 延伸阅读 → 紫色（`.sr`）

#### 3.6.2 引用块（Citation Block）
- 浅色背景（`--acc-w`），圆角，细边框
- 标题栏：📖 图标 + 书名 + 章节名
- 正文区：
  - 原文引用：左边框强调，灰色背景
  - "释义"标签 + 释义文字

#### 3.6.3 深度阐释条目
- 小圆点前缀（`::before` 伪元素）
- 粗体标题 + 正文段落
- 正文缩进左对齐

### 3.7 参考文献区
- 浅灰背景（`#f8f5f0`），圆角边框
- 编号圆形徽标（棕色底白字）
- 书名粗体 + 摘要（最多3行截断）
- 最后一项无边线

### 3.8 加载动画
- 三个圆点交替缩放 + 透明度变化
- 圆点颜色 `--acc-s`
- 动画 `1.2s ease-in-out infinite`

### 3.9 输入栏（Input Bar）
- `position: sticky; bottom: 0`
- 半透明渐变上边缘（制造浮起效果）
- 输入框：圆角 `20px`，聚焦时主题色边框 + 发光
- 发送按钮：渐变背景，hover 上移 + 加深阴影
- 输入框占位符颜色 `--ink-p`

## 四、JavaScript 交互逻辑

### 4.1 流式打字效果（typeWriter）
- 逐字符输出，带标点停顿节奏：
  - 普通字符：`35ms`
  - 空格：`1.5×`
  - 逗号/冒号/分号：`4×`
  - 句号/问号/感叹号：`6×`
  - 换行：`8×`
- 末尾带闪烁光标（`typing-cursor`）
- 打字完成后移除光标，替换为完全格式化的 HTML

### 4.2 思考小窗更新
- 默认显示 5 步分析流程
- 可折叠/展开（`toggle` 切换）
- 内容动态更新（`updateText` 方法）

### 4.3 回答渲染流程
1. 原始文本按 emoji 标记分割段落
2. 检测每段是否为引用块（`**出处：**`）
3. 非引用块按节标题渲染或直接作为段落
4. 渲染完成后追加参考文献区域

### 4.4 发送流程
1. 禁用输入框和按钮（防止重复提交）
2. 显示加载动画
3. POST 请求 `/rag/query`
4. 成功：移除加载 → 渲染回答（含思考窗 + 打字效果）
5. 失败：显示错误消息

## 五、响应式设计

- 所有尺寸使用 `clamp()` 函数实现流体缩放
- 移动端适配：
  - 设置 `maximum-scale=1.0, user-scalable=no`
  - 最小字号 `14px`，最大 `17px`
  - 内边距随视口宽度变化
- 桌面端：最大内容宽度 `740px`

## 六、动画约定

| 动画 | 触发时机 | 时长 | 缓动函数 |
|------|---------|------|---------|
| `fU`（淡入上滑） | 消息气泡出现 | `0.4s` | `ease-out` |
| `fadeIn` | 副标题文字 | `1.2s` | `ease` |
| `ldB`（加载点跳动） | 加载中 | `1.2s` | `ease-in-out` |
| `blink`（光标闪烁） | 打字中 | `0.8s` | `step-end` |
| 思考窗展开/收起 | 点击标题 | `0.4s` | `cubic-bezier(.4,0,.2,1)` |

docs/
  UI_DESIGN.md               # 本文件，设计规范文档

## 八、修改指南

### 修改前端样式
1. 编辑 `outputs/_gen_frontend_v5.py` 中的 CSS 变量
2. 重新运行生成脚本
3. 重启服务

### 修改回答格式
1. 编辑 `src/rag/rag_engine.py` 中的 `SYSTEM_PROMPT` 字符串
2. 重启服务

### 新增组件
1. 在 CSS 中添加对应样式类
2. 在 JS 中添加渲染函数
3. 在 `renderAnswer` 中注册新组件的解析逻辑
4. 更新本文档
